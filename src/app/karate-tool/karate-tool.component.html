Response Comparison Tool

How to compare response?

Compare results easily using karate!

Why Karate?

How to make sure test results after migration matches the results with Tibco. Each service may have multiple test cases so comparing them manually using tools like beyond compare is cumbersome job.


How it works?

Karate framework is already used for functional/regression automated test. It is extended by adding capability to execute test against both Tibco, SpringBoot and compare the responses.

In the application, we have to give both tibco and spring boot endpoint along with request body.

Along with this, we can set our filter for comparison as well. Based on that filter the comparion logic will work.

The application creates both spring boot and tibco responses as a xml files.

It will compare the both responses and show what is the difference between them.

Based on the differences in result, we can triage further.

Select service name and environment to compare responses!

If you want to run a particular service in a particular environment, select the values in the following table to compare the responses.

1. Select the service name

2. Select the environmet

Run the pipeline to see the karate report!

1. Run the pipeline in gitlab to build the application.

2. Once the application is build successfully, you can compare spring boot and tibco responses.

3. After comparison, the HTML report will be generated to download.



Advantages

1. It reduces the effort to compare the differences.

2. Savings- 100% automation for results comparison.

3. Benefits us during multiple test cycles.

4. It provides ability to track comparison reports of previous test cycles
